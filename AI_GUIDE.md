# BOOONG Design System AI Guide

Use this guide when an AI assistant works on a BOOONG-based project.

## Rules

- Use BOOONG design tokens before introducing new raw colors, spacing, radii, shadows, or font stacks.
- Keep these tokens stable unless a design-system release intentionally changes them: `--brand`, `--brand-deep`, `--brand-soft`, `--brand-ink`, `--bg`, `--text`, `--border`, `--radius`, `--space-*`, `--font-*`.
- Reuse existing component classes before creating project-specific variants.
- Project CSS should define product-specific layout, page composition, and domain visuals.
- Do not redefine brand colors, typography scale, spacing scale, or radii inside consumer projects.
- Preserve legacy classes generated by BOOONG dashboard JavaScript, including `.block`, `.callout`, `.question`, `.case`, `.answer`, and `.material`.

## Static HTML Usage

```html
<link rel="stylesheet" href="https://yadoran-2025.github.io/booong-design-system/releases/v1.0.0/booong.css">
<link rel="stylesheet" href="./css/project.css">
```

## npm Usage

```js
import "@yadoran-2025/booong-design-system/dist/booong.css";
import "./project.css";
```
